# Plan: Add Native Fish Function Tests with Fishtape

## Context

The dotfiles repo has ShellSpec tests that syntax-check `.fish` files but no behavioral tests for the ~40 fish functions. This plan adds native fish tests using **fishtape 3** — a TAP-based test runner written in pure Fish — to test the actual business logic: env file parsing, flake.nix traversal, git command sequencing, and tmux pane search.

## Fishtape 3 API

```fish
@test "description" [actual] operator expected
# actual can be a command substitution
@test "parses KEY=VALUE" $FOO = bar
@test "exits with 0" (some_command >/dev/null) $status -eq 0
@test "file exists" -f /path/to/file
```

Each test file runs in its own fish shell — global env changes stay isolated.

## What to Test

### `_hm_load_env_file` — pure parsing logic, no mocks needed
- `KEY=VALUE` → env var exported
- `"quoted value"` → quotes stripped
- `'single quoted'` → quotes stripped
- `export KEY=VALUE` prefix stripped
- `# comment` lines skipped
- blank lines skipped
- returns cleanly when no env file exists

### `_dev_function` — flake.nix traversal (mock `nix`)
- cwd has `flake.nix` → `echo "Entering devshell in <cwd>"`
- parent has `flake.nix` → `echo "Entering devshell in <parent>"`
- no `flake.nix` anywhere → `echo "Entering devshell in ~/dotfiles"`

### `_tsh_function` — early return + no mocks
- pane dir missing → prints `"No pane content store found at ..."`, exits 0

### `_gco_function` — git sequence (mock `git`)
- calls `git symbolic-ref refs/remotes/origin/HEAD` → reads branch
- then `git checkout <branch>` → switches
- then `git pull` → updates

### `_grco_function` — git reset sequence (mock `git`)
- calls `git fetch origin <branch>`
- then `git checkout <branch>`
- then `git reset --hard origin/<branch>`
