  install nixos installer w/ full disk encryption how to

  Here's a practical, up-to-date (as of 2025–2026) way to install NixOS from the live USB with full disk encryption using LUKS (most common & well-supported approach).

  There are two main paths:

  Easiest — use the graphical Calamares installer (available in the plasma / gnome ISOs) → select "Encrypt" during partitioning
  Manual (recommended for control, flakes, Btrfs/ZFS, custom LVM, TPM auto-unlock later, etc.)
  Most experienced NixOS users do the manual method because it gives full control over the config from the beginning.

  Quick Recommendation Table (2025–2026 reality)
  Goal  Best Method in 2026     Approx. Difficulty
  Fastest / beginner    Graphical installer + Encrypt option    Easy
  Good control + LVM    Manual + LVM-on-LUKS    Medium
  Btrfs + snapshots     Manual + disko + Btrfs-on-LUKS  Medium+
  TPM2 auto-unlock (no password)        Manual + systemd-cryptenroll    Medium–Hard
  YubiKey / FIDO2 unlock        Manual + YubiKey PBA    Hard
  Modern Recommended Manual Method (LUKS + LVM or plain ext4/btrfs)
  Boot the latest minimal or plasma ISO → get internet → become root (sudo -i)

  0. Preparation
  # Check disks (be VERY careful!)
  lsblk -f
  nvme list    # or just ls /dev/nvme* /dev/sd*

  # Most common variables (ADJUST THESE!)
  DISK="/dev/nvme0n1"           # ← change this!!!
  HOST="mymachine"              # used in configuration.nix
  1. Partitioning (UEFI — most laptops/desktops 2025+)
  parted --script $DISK                    \
    mklabel gpt                            \
    mkpart ESP     fat32        1MiB 1GiB  \
    set 1 esp on                           \
    mkpart LUKS    1GiB         100%       # rest of disk
  → You now have /dev/nvme0n1p1 (EFI) and /dev/nvme0n1p2 (to be encrypted)

  2. Format EFI partition
  mkfs.fat -F 32 -n BOOT /dev/nvme0n1p1
  3. Create LUKS container (2025+ defaults are good)
  cryptsetup --type luks2 \
    --cipher aes-xts-plain64 --key-size 512 --hash sha512 \
    --pbkdf argon2id --iter-time 2000 \
    luksFormat /dev/nvme0n1p2

  # You will be asked for passphrase twice (choose strong one!)

  # Open it (choose any mapper name you like — "cryptroot" is common)
  cryptsetup open /dev/nvme0n1p2 cryptroot
  → unlocked device now lives at /dev/mapper/cryptroot

  4a. Option A – Simple (no LVM, just one ext4 or btrfs partition)
  # ext4 example
  mkfs.ext4 -L nixos /dev/mapper/cryptroot

  # OR better — btrfs (snapshots, compression, etc.)
  mkfs.btrfs -L nixos -f /dev/mapper/cryptroot
  Mount:

  mount /dev/mapper/cryptroot /mnt
  mkdir -p /mnt/boot
  mount /dev/nvme0n1p1 /mnt/boot
  4b. Option B – LVM on LUKS (still very popular in 2026)
  pvcreate /dev/mapper/cryptroot
  vgcreate vg /dev/mapper/cryptroot

  lvcreate -L  60G -n root vg     # adjust sizes
  lvcreate -L   8G -n swap vg     # or more/less
  # optional: lvcreate -l 100%FREE -n home vg

  mkfs.ext4 -L root /dev/vg/root
  mkswap     -L swap /dev/vg/swap
  # mkfs.ext4 -L home /dev/vg/home   (if you made home)

  mount  /dev/vg/root /mnt
  mkdir -p /mnt/boot
  mount /dev/nvme0n1p1 /mnt/boot
  swapon /dev/vg/swap
  5. Generate & tweak configuration
  nixos-generate-config --root /mnt

  # Now edit
  nano /mnt/etc/nixos/configuration.nix
  Important minimal changes for LUKS:

  { config, pkgs, ... }:

  {
    # ──── required for decryption ────────────────────────────────
    boot.initrd.availableKernelModules = [ "xhci_pci" "ahci" "nvme" "usbhid" "usb_storage" "sd_mod" ];
    boot.initrd.kernelModules = [ "dm-snapshot" ];   # if you use LVM + snapshots later

    boot.initrd.luks.devices."cryptroot" = {
      device = "/dev/nvme0n1p2";   # or "/dev/disk/by-uuid/…" (more robust)
      # fallbackToPassword = true; # optional
    };

    # If using LVM
    boot.initrd.luks.devices."cryptroot".preLVM = true;   # most people need this

    fileSystems."/" = {
      device = "/dev/vg/root";     # or "/dev/mapper/cryptroot" if no LVM
      fsType = "ext4";             # or "btrfs"
    };

    fileSystems."/boot" = {
      device = "/dev/nvme0n1p1";
      fsType = "vfat";
    };

    swapDevices = [ { device = "/dev/vg/swap"; } ];   # if you made swap

    # Recommended 2025+ additions
    boot.loader.systemd-boot.enable = true;
    boot.loader.efi.canTouchEfiVariables = true;

    # networking, timezone, users, desktop, etc...
  }
  6. Install
  nixos-install --no-root-passwd   # or with --root-password if you want

  # Reboot (remove USB)

  Quick disko + flakes modern style (very popular 2025–2026)
  Many people now use https://github.com/nix-community/disko

  Example minimal disko config → put in disko-config.nix → run

  nix --extra-experimental-features "nix-command flakes" \
    run github:nix-community/disko -- --mode disko disko-config.nix

  Most common gotchas in 2026:

  Wrong UUID in boot.initrd.luks.devices
  Missing kernel modules → no decryption prompt appears
  preLVM = true; needed when using LVM
  For TPM auto-unlock → add systemd-cryptenroll step after first boot

  The debug: /dev/nvme0n1p1 is not an HFS+ partition: exiting line is very likely harmless probe noise, not the real failure. That exact wording matches os-prober trying its macOS
  checker against your EFI partition. nvme0n1p1 is usually the EFI partition, which should be vfat, not HFS+.

  If you already created the encrypted partition and want to keep it, do not run any of these again on that disk: mklabel, mkpart, mkfs.*, luksFormat. Those would overwrite it.

  Use the existing layout instead:

  lsblk -f
  blkid

  # assuming the encrypted partition you want to keep is p2
  cryptsetup open /dev/nvme0n1p2 cryptroot

  # if cryptroot contains a filesystem directly:
  mount /dev/mapper/cryptroot /mnt

  # if cryptroot contains LVM instead:
  vgchange -ay
  mount /dev/vg/root /mnt
  swapon /dev/vg/swap   # if you have swap

  Then mount the existing EFI partition without touching it:

  mkdir -p /mnt/boot
  mount /dev/nvme0n1p1 /mnt/boot
  nixos-generate-config --root /mnt

  Your NixOS config should reference the existing encrypted partition, preferably by UUID:

  boot.loader.systemd-boot.enable = true;
  boot.loader.efi.canTouchEfiVariables = true;

  boot.initrd.luks.devices.cryptroot.device =
    "/dev/disk/by-uuid/<UUID-of-nvme0n1p2>";

  fileSystems."/boot" = {
    device = "/dev/disk/by-uuid/<UUID-of-nvme0n1p1>";
    fsType = "vfat";
  };

  Then set / to either:

  - /dev/mapper/cryptroot if the LUKS device holds the filesystem directly
  - /dev/vg/root if you used LVM inside LUKS

  If the partition you want to “keep” is macOS FileVault/APFS, that is different: NixOS cannot install inside that encrypted macOS partition. In that case, keep it untouched and
  create a separate partition for NixOS, reusing only the EFI partition.
