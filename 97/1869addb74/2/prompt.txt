theme is switched to transparent, but the Home Manager module at config/opencode/default.nix only installs ~/.config/opencode/opencode.jsonc (it does not install ~/.config/opencode/themes/transparent.json). In a Nix/Home Manager setup this will leave the config referencing a theme that isn’t present, and opencode may fall back or error on startup. Consider also deploying the themes/ directory (or at least transparent.json) alongside the config, or keep using a built-in theme name.

Suggested change
