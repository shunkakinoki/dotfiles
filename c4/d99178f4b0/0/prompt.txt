# Plan: Add Native Fish Function Tests with Fishtape

## Context

The dotfiles repo has ShellSpec tests that syntax-check `.fish` files but no behavioral tests for the ~40 fish functions. This plan adds native fish tests using **fishtape 3** — a TAP-based test runner written in pure Fish — to test the actual business logic: env file parsing, flake.nix traversal, git command sequencing, and tmux pane search.

## Fishtape 3 API

```fish
@test "description" [actual] operator expected
# actual can be a command substitution
@test "parses KEY=VALUE" $FOO = bar
@test "exits with 0" (some_command >/dev/null) $status -eq 0
@test "file exists" -f /path/to/file
```

Each test file runs in its own fish shell — global env changes stay isolated.

## What to Test

### `_hm_load_env_file` — pure parsing logic, no mocks needed
- `KEY=VALUE` → env var exported
- `"quoted value"` → quotes stripped
- `'single quoted'` → quotes stripped
- `export KEY=VALUE` prefix stripped
- `# comment` lines skipped
- blank lines skipped
- returns cleanly when no env file exists

### `_dev_function` — flake.nix traversal (mock `nix`)
- cwd has `flake.nix` → `echo "Entering devshell in <cwd>"`
- parent has `flake.nix` → `echo "Entering devshell in <parent>"`
- no `flake.nix` anywhere → `echo "Entering devshell in ~/dotfiles"`

### `_tsh_function` — early return + no mocks
- pane dir missing → prints `"No pane content store found at ..."`, exits 0

### `_gco_function` — git sequence (mock `git`)
- calls `git symbolic-ref refs/remotes/origin/HEAD` → reads branch
- then `git checkout <branch>` → switches
- then `git pull` → updates

### `_grco_function` — git reset sequence (mock `git`)
- calls `git fetch origin <branch>`
- then `git checkout <branch>`
- then `git reset --hard origin/<branch>`

## Implementation

### `devenv.nix`

Add two packages:

```nix
pkgs.fish
(pkgs.writeShellScriptBin "fishtape" ''
  exec ${pkgs.fish}/bin/fish \
    -C "source ${pkgs.fishPlugins.fishtape_3.src}/functions/fishtape.fish" \
    "$@"
'')
```

This makes `fishtape` a shell command that executes test files inside fish with fishtape pre-loaded.

### `spec/fish/_hm_load_env_file_test.fish`

```fish
set fn (status dirname)/../../home-manager/programs/fish/functions
source $fn/_hm_load_env_file.fish

# ── KEY=VALUE ──────────────────────────────────────────────
set tmpdir (mktemp -d)
printf "FOO=bar\n" > $tmpdir/t.env
set -x DOTFILES_ENV_FILE $tmpdir/t.env
_hm_load_env_file
@test "parses KEY=VALUE" $FOO = bar

# ── double-quoted value ────────────────────────────────────
printf 'QVAL="hello world"\n' > $tmpdir/t.env
set -x DOTFILES_ENV_FILE $tmpdir/t.env
_hm_load_env_file
@test "strips double quotes" $QVAL = "hello world"

# ── export prefix ─────────────────────────────────────────
printf "export EXPKEY=expval\n" > $tmpdir/t.env
set -x DOTFILES_ENV_FILE $tmpdir/t.env
_hm_load_env_file
@test "strips export prefix" $EXPKEY = expval

# ── comment and blank lines ───────────────────────────────
printf "# comment\n\nBARK=baz\n" > $tmpdir/t.env
set -x DOTFILES_ENV_FILE $tmpdir/t.env
_hm_load_env_file
@test "skips comments" $BARK = baz

# ── missing env file ───────────────────────────────────────
set -e DOTFILES_ENV_FILE
set -x HOME $tmpdir
@test "returns 0 when no file" (_hm_load_env_file; echo done) = done

rm -rf $tmpdir
```

### `spec/fish/_dev_function_test.fish`

```fish
set fn (status dirname)/../../home-manager/programs/fish/functions
source $fn/_dev_function.fish

set tmpdir (mktemp -d)
mkdir -p $tmpdir/child/grandchild

# Mock nix so devshell doesn't actually launch
function nix; end

# ── cwd has flake.nix ─────────────────────────────────────
touch $tmpdir/child/flake.nix
@test "uses cwd" (cd $tmpdir/child; _dev_function) = "Entering devshell in $tmpdir/child"

# ── grandchild → walks up to parent ──────────────────────
@test "walks up to parent" (cd $tmpdir/child/grandchild; _dev_function) = "Entering devshell in $tmpdir/child"

# ── no flake.nix → falls back to ~/dotfiles ──────────────
rm $tmpdir/child/flake.nix
set -x HOME $tmpdir
@test "falls back to dotfiles" (cd $tmpdir/child/grandchild; _dev_function) = "Entering devshell in $tmpdir/dotfiles"

rm -rf $tmpdir
```

### `Makefile` additions

```makefile
.PHONY: fish-test
fish-test: ## Run fish function tests using fishtape.
    @echo "🐟 Running fish function tests..."
    @fishtape spec/fish/*_test.fish

.PHONY: fish-test-dev
fish-test-dev: ## Run fish tests inside the Nix dev shell (mirrors CI).
    @echo "🐟 Running fish function tests inside the Nix dev shell..."
    @DEVENV_ROOT=$(CURDIR) $(NIX_ALLOW_UNFREE) $(NIX_EXEC) develop $(NIX_FLAGS) .# --command $(MAKE) fish-test
```
